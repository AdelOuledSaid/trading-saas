Python Setup for Trading Analysis

Your quantitative environment in 20 minutes — no prior coding required

Python is the standard language for quantitative finance. You do not need to become a programmer — you need to read, modify and run existing scripts that automate analysis tasks that would take hours manually. This module gets your environment running and tests it with your first real analysis.

Setup in 4 steps

  • Step 1 — Install Python: Download Python 3.11+ from python.org. During install, check "Add Python to PATH".
  • Step 2 — Install libraries: Open terminal/command prompt and run: pip install pandas numpy yfinance matplotlib scipy
  • Step 3 — Install VS Code: Free code editor from code.visualstudio.com — install the Python extension.
  • Step 4 — Test with first script: Run the code below to download and plot BTC daily data.
import yfinance as yf import matplotlib.pyplot as plt # Download BTC daily data — last 2 years btc = yf.download("BTC-USD", period="2y", interval="1d") # Plot closing price plt.figure(figsize=(12, 5)) plt.plot(btc['Close'], color='#06b6d4', linewidth=1.5) plt.title('BTC/USD — Daily Close') plt.xlabel('Date') plt.ylabel('Price USD') plt.grid(alpha=0.2) plt.show() print(f"Data shape: {btc.shape}") print(btc.tail())

Manual backtesting (Level 5) is limited to ~200 trades in a few months of data. Python backtesting processes 5 years of data, 1,000+ occurrences, across multiple instruments, in seconds. This is not automation for its own sake — it is the difference between 80% confidence and 99% statistical confidence in your edge.

Sharpe & Sortino Ratio

Measuring return per unit of risk — the institutional standard

Win rate and expectancy measure whether a strategy makes money. Sharpe and Sortino ratios measure whether it makes money efficiently relative to the risk taken — which is the institutional standard for evaluating any trading system.

Sharpe Ratio

Sharpe = (Average return − Risk-free rate) ÷ Standard deviation of returns. Measures excess return per unit of total volatility. A Sharpe above 1.0 is considered good; above 2.0 is excellent. A strategy with 15% annual return and 5% volatility (Sharpe 3.0) is superior to one with 30% return and 25% volatility (Sharpe 1.2).

Sortino Ratio

Sortino = (Average return − Risk-free rate) ÷ Downside deviation. Unlike Sharpe, it only penalizes downside volatility — upside variance is not punished. For trading strategies with asymmetric return distributions (few large winners), Sortino is typically the more relevant metric.

# Calculate Sharpe and Sortino from your trade R-multiples import numpy as np # Your trade results in R (replace with your actual data) trades = [1.8, -1.0, 2.3, -1.0, 1.5, 3.1, -1.0, -1.0, 2.8, 1.2] returns = np.array(trades) rf = 0 # risk-free rate in R terms sharpe = (np.mean(returns) - rf) / np.std(returns) downside = returns[returns < 0] sortino = (np.mean(returns) - rf) / np.std(downside) if len(downside) > 0 else np.inf print(f"Sharpe Ratio: {sharpe:.2f}") print(f"Sortino Ratio: {sortino:.2f}") print(f"Expectancy: {np.mean(returns):.3f}R per trade")

Maximum Drawdown Analysis

Understanding the true risk of your strategy

Maximum drawdown (MDD) is the largest peak-to-trough decline in your equity curve. It is the most psychologically significant metric for discretionary traders — because it defines the worst period you will experience if you run the strategy long enough.

# Calculate maximum drawdown from equity curve import numpy as np import pandas as pd # Cumulative R-multiples (replace with your data) equity = [0, 1.8, 0.8, 3.1, 2.1, 3.6, 6.7, 5.7, 4.7, 7.5, 8.7] equity_series = pd.Series(equity) rolling_max = equity_series.cummax() drawdown = equity_series - rolling_max max_dd = drawdown.min() max_dd_pct = (max_dd / rolling_max[drawdown.idxmin()]) * 100 print(f"Maximum Drawdown: {max_dd:.2f}R") print(f"Max Drawdown %: {max_dd_pct:.1f}%") print(f"Drawdown occurred at trade: {drawdown.idxmin()}")
The psychological vs statistical MDD

A statistically acceptable MDD of –10R sounds manageable on paper. Experiencing it in real trading — where each loss feels permanent — is psychologically very different. Before deploying a strategy, simulate the worst historical drawdown period mentally: "If I had 8 consecutive losses at my planned risk, would I continue the strategy?" If the honest answer is no, reduce position sizing until the answer becomes yes.

Win Distribution Analysis

Understanding the shape of your returns — not just the average

Average expectancy tells you the mean outcome. Win distribution tells you the shape — are wins clustered near 1R with occasional 3R outliers, or is the distribution more uniform? This matters for understanding variance, streak risk and psychological sustainability.

# Visualize your win distribution import matplotlib.pyplot as plt import numpy as np trades = [-1,-1,1.8,-1,2.3,3.1,-1,1.5,-1,2.8,4.2,-1,-1,1.2,2.0,-1,3.5,-1,1.9,-1] winners = [t for t in trades if t > 0] losers = [t for t in trades if t < 0] fig, axes = plt.subplots(1, 2, figsize=(12, 4)) axes[0].hist(winners, bins=10, color='#22c55e', alpha=0.7, edgecolor='white') axes[0].set_title('Winner Distribution (R)') axes[0].set_xlabel('R-multiple') axes[1].hist(losers, bins=10, color='#ef4444', alpha=0.7, edgecolor='white') axes[1].set_title('Loser Distribution (R)') axes[1].set_xlabel('R-multiple') for ax in axes: ax.set_facecolor('#111520') ax.tick_params(colors='#c8d0e0') plt.tight_layout() plt.show() print(f"Avg winner: {np.mean(winners):.2f}R | Avg loser: {np.mean(losers):.2f}R") print(f"Largest winner: {max(winners):.2f}R | Largest loser: {min(losers):.2f}R")
Fat tail risk

A heavy left tail (losses significantly larger than –1R) indicates poor stop management — stops are being moved or missed. This single issue can destroy a mathematically positive expectancy strategy. If your loser distribution shows any result beyond –2R, investigate every instance for execution failures.

06

Monte Carlo Simulation

Testing strategy robustness against random outcome sequences

A backtest shows you one specific historical sequence of outcomes. Monte Carlo simulation shuffles those same outcomes thousands of times to answer: "What is the range of possible equity curves if my edge is real but outcomes occur in different orders?" This reveals the true worst-case drawdown and the probability of ruin at different risk levels.

import numpy as np import matplotlib.pyplot as plt # Your historical trade results in R trades = [-1,-1,1.8,-1,2.3,3.1,-1,1.5,-1,2.8,4.2,-1,-1,1.2,2.0,-1,3.5,-1,1.9,-1] n_simulations = 1000 n_trades = 100 # forward-looking all_equity = [] for _ in range(n_simulations): sample = np.random.choice(trades, size=n_trades, replace=True) equity = np.cumsum(sample) all_equity.append(equity) all_equity = np.array(all_equity) # Plot plt.figure(figsize=(12, 5)) for eq in all_equity[:100]: plt.plot(eq, alpha=0.05, color='#06b6d4') plt.plot(np.median(all_equity, axis=0), color='white', linewidth=2, label='Median') plt.axhline(0, color='#ef4444', linestyle='--', alpha=0.5) plt.title(f'Monte Carlo — {n_simulations} simulations of {n_trades} trades') plt.xlabel('Trade number') plt.ylabel('Cumulative R') plt.legend() plt.tight_layout() plt.show() worst_drawdown = np.min(np.min(all_equity, axis=1)) pct_profitable = np.sum(all_equity[:,-1] > 0) / n_simulations * 100 print(f"Worst case drawdown: {worst_drawdown:.1f}R") print(f"Profitable after {n_trades} trades: {pct_profitable:.0f}% of simulations")
Interpreting Monte Carlo results

If 90%+ of simulations are profitable after 100 trades, your edge is robust. If the worst-case drawdown in any simulation exceeds your psychological tolerance, reduce position sizing until it doesn't. The Monte Carlo output is the statistical answer to "how bad can it get?" — far more honest than the average backtest line.

Automated Signal Detection

Using Python to scan for setup conditions across multiple instruments

Once your edge is defined with precise conditions, Python can scan historical data for every occurrence automatically — removing the selection bias of manual scanning and processing years of data in seconds.

import yfinance as yf import pandas as pd import numpy as np def detect_signals(symbol, period="1y"): """Detect simple pullback-to-EMA signals as an example framework.""" df = yf.download(symbol, period=period, interval="1d", progress=False) # Calculate EMAs df['ema_20'] = df['Close'].ewm(span=20).mean() df['ema_50'] = df['Close'].ewm(span=50).mean() # Signal: price pulls back to EMA20 in uptrend (EMA20 > EMA50) df['uptrend'] = df['ema_20'] > df['ema_50'] df['near_ema20'] = abs(df['Close'] - df['ema_20']) / df['ema_20'] < 0.01 # within 1% df['signal'] = df['uptrend'] & df['near_ema20'] signals = df[df['signal']].copy() return signals[['Close', 'ema_20', 'ema_50']].round(2) # Scan multiple instruments symbols = ['BTC-USD', 'ETH-USD', 'SOL-USD'] for sym in symbols: sigs = detect_signals(sym) print(f"\n{sym}: {len(sigs)} signals found") print(sigs.tail(3))
Automation vs discretion

Automated signal detection is a scanning tool, not an execution tool. At Level 6, signals are identified programmatically but executed with the structural and confluence filters from Levels 2–5. Full automation (algorithmic trading) introduces execution infrastructure requirements beyond the scope of the Academy curriculum.

Statistical Hypothesis Testing

Proving your edge is real, not random

The t-test allows you to statistically determine whether your strategy's positive expectancy is significantly different from zero — or whether it could plausibly be explained by random chance.

from scipy import stats import numpy as np # Your backtest trade results in R trades = [-1,-1,1.8,-1,2.3,3.1,-1,1.5,-1,2.8,4.2,-1,-1,1.2,2.0,-1,3.5,-1,1.9,-1] # One-sample t-test against null hypothesis (expectancy = 0) t_stat, p_value = stats.ttest_1samp(trades, 0) print(f"Sample size: {len(trades)}") print(f"Mean expectancy: {np.mean(trades):.3f}R per trade") print(f"T-statistic: {t_stat:.3f}") print(f"P-value: {p_value:.4f}") if p_value < 0.05: print("→ Edge is STATISTICALLY SIGNIFICANT (p < 0.05)") print(" Less than 5% probability this result is random luck") else: print("→ Edge is NOT statistically significant — need more data or stronger edge")
Interpreting p-value

p < 0.05 means there is less than 5% probability that your results occurred by random chance. This is the standard threshold for statistical significance. p < 0.01 means less than 1% probability — very high confidence. If your p-value is above 0.05 with 100+ trades, your edge is not statistically proven — continue collecting data or revise the strategy.

import yfinance as yf import seaborn as sns import matplotlib.pyplot as plt # Download price data for key assets assets = { 'BTC': 'BTC-USD', 'ETH': 'ETH-USD', 'SOL': 'SOL-USD', 'US100': 'QQQ', 'Gold': 'GLD', 'DXY': 'UUP' } prices = {} for name, symbol in assets.items(): data = yf.download(symbol, period="1y", interval="1d", progress=False) prices[name] = data['Close'] import pandas as pd df = pd.DataFrame(prices).dropna() returns = df.pct_change().dropna() corr = returns.corr() # Plot heatmap plt.figure(figsize=(8, 6)) sns.heatmap(corr, annot=True, fmt='.2f', cmap='RdYlGn', center=0, vmin=-1, vmax=1, annot_kws={"size": 10}) plt.title('Asset Return Correlation Matrix (1Y daily)') plt.tight_layout() plt.show() print("\nHighest correlations:") print(corr.unstack().sort_values(ascending=False).drop_duplicates().head(8))

Regime Classification with Data

Algorithmically identifying market regimes from price data

Level 4 taught you to identify market regimes manually. Level 6 teaches you to classify them algorithmically — creating a data-driven regime filter that can be applied consistently across instruments and timeframes.

import yfinance as yf import pandas as pd import numpy as np def classify_regime(symbol, period="1y"): df = yf.download(symbol, period=period, interval="1d", progress=False) close = df['Close'] # Simple regime classification using EMAs and ATR ema20 = close.ewm(span=20).mean() ema50 = close.ewm(span=50).mean() ema200 = close.ewm(span=200).mean() high_low_range = df['High'] - df['Low'] atr = high_low_range.rolling(14).mean() atr_pct = atr / close * 100 def get_regime(row): if row['ema20'] > row['ema50'] > row['ema200']: return 'EXPANSION' if row['atr_pct'] < 3 else 'VOLATILE_BULL' elif row['ema20'] < row['ema50'] < row['ema200']: return 'BREAKDOWN' if row['atr_pct'] > 4 else 'DOWNTREND' elif abs(row['ema20'] - row['ema50']) / row['ema50'] < 0.02: return 'COMPRESSION' else: return 'TRANSITION' df['ema20'] = ema20; df['ema50'] = ema50; df['ema200'] = ema200 df['atr_pct'] = atr_pct df['regime'] = df.apply(get_regime, axis=1) print(f"\n{symbol} — Regime distribution (last {period}):") print(df['regime'].value_counts()) print(f"\nCurrent regime: {df['regime'].iloc[-1]}") return df classify_regime('BTC-USD')

Combining quantitative rules with structural discretion

Pure discretionary trading loses the consistency advantages of quantitative rule-following. The hybrid framework — the approach used by most successful individual traders at Level 6+ — combines systematic filtering with discretionary confirmation.
